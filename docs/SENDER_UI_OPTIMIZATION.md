# 报文发送界面优化说明

## 功能概述

报文发送功能支持两种模式：
1. **模板模式**：选择已保存的报文模板，自动渲染后发送
2. **自定义模式**：直接输入报文内容，无需创建模板

## 界面布局

### 整体结构
采用左右分栏布局，提升用户体验：

```
┌─────────────────────────────────────────────────────┐
│                    页面标题栏                        │
├──────────────┬──────────────────────────────────────┤
│              │                                      │
│   左侧配置   │         右侧预览和响应               │
│   (480px)    │         (自适应宽度)                 │
│              │                                      │
│  - 模板选择  │  - 请求预览                          │
│  - 协议Tab   │  - 响应结果                          │
│  - 参数配置  │                                      │
│  - 发送按钮  │                                      │
│              │                                      │
└──────────────┴──────────────────────────────────────┘
```

### 左侧配置区域 (480px 固定宽度)

1. **报文来源选择**
   - 下拉选择框，包含"自定义输入"选项和所有可用模板
   - 选择"自定义输入"：右侧显示文本输入框
   - 选择模板：右侧显示模板渲染预览
   - 自动触发预览更新

2. **协议配置 (Tab 形式)**
   - 使用 Tabs 组件展示所有支持的协议
   - 每个协议一个 Tab，点击切换
   - Tab 内容区显示该协议的参数配置表单
   - 参数根据类型自动渲染（文本框/下拉框）
   - 必填参数标注红色星号

3. **发送按钮**
   - 固定在左侧底部
   - 全宽按钮，带加载状态
   - 未选择模板或协议时禁用

### 右侧预览和响应区域 (自适应宽度)

1. **请求预览 / 自定义输入**
   - **模板模式**：
     - 选择模板后自动渲染并显示
     - 显示渲染后的完整报文内容
     - 支持手动刷新按钮
     - 渲染中显示加载状态
   - **自定义模式**：
     - 使用 CodeMirror 编辑器，支持语法高亮
     - 支持 JSON 和 XML 格式切换
     - 提供格式化按钮，自动美化代码
     - 提供快速粘贴按钮，一键粘贴剪贴板内容
     - 等宽字体显示，行号、代码折叠、括号匹配
   - 未选择时显示提示占位符

2. **响应结果**
   - 发送成功后显示
   - 顶部显示状态（成功/失败）、状态码、耗时
   - 成功时使用 CodeMirror 显示响应内容
   - 自动检测响应格式（JSON/XML）并应用语法高亮
   - 后端自动格式化响应内容，前端只读显示
   - 失败时显示错误信息（红色高亮）
   - 支持滚动查看长内容

## 交互流程

### 模板模式
1. 用户选择报文模板 → 自动渲染预览
2. 用户切换协议 Tab → 显示对应参数配置
3. 用户填写参数 → 实时验证必填项
4. 用户点击发送 → 显示加载状态 → 展示响应结果

### 自定义模式
1. 用户选择"自定义输入" → 右侧显示文本输入框
2. 用户输入报文内容 → 实时验证非空
3. 用户切换协议 Tab → 显示对应参数配置
4. 用户填写参数 → 实时验证必填项
5. 用户点击发送 → 显示加载状态 → 展示响应结果

## 优化亮点

1. **双模式支持**：模板模式和自定义模式灵活切换
2. **Tab 展示协议**：清晰直观，易于扩展新协议
3. **左右分栏**：配置和预览分离，避免滚动查找
4. **自动预览**：选择模板后立即渲染，所见即所发
5. **代码编辑器**：自定义模式使用 CodeMirror，支持语法高亮和格式化
6. **快速操作**：提供格式化和快速粘贴按钮，提升输入效率
7. **格式切换**：自定义模式支持 JSON/XML 格式切换
8. **响应格式化**：自动检测并格式化响应内容，提升可读性
9. **语法高亮**：请求和响应都使用 CodeMirror 显示，支持语法高亮
10. **固定宽度左侧**：保持配置区域稳定，不受内容影响
11. **响应保留**：发送后响应结果保留在页面，便于对比
12. **状态反馈**：加载、成功、失败状态清晰可见

## 技术实现

### 前端
- 使用 shadcn/ui Tabs 组件实现协议切换
- 使用 CodeMirror 编辑器实现自定义输入，支持语法高亮
- 集成 @codemirror/lang-json 和 @codemirror/lang-xml 语言支持
- 使用 oneDark 主题提供舒适的编辑体验
- 左侧固定 480px 宽度，右侧 flex-1 自适应
- 使用 useEffect 监听模板选择，自动触发预览
- 协议切换时自动填充默认参数值
- 响应结果使用条件渲染，不影响布局高度
- 自定义模式和模板模式状态独立管理
- 格式化功能调用后端 API，保证格式一致性
- 快速粘贴功能使用 Clipboard API 读取剪贴板
- 响应内容使用 CodeMirror 只读模式显示
- 自动检测响应格式并应用对应的语法高亮
- 响应编辑器禁用活动行高亮，优化只读体验

### 后端
- `SendMessageRequest` 支持 `templateId` 和 `customContent` 两个可选字段
- `MessageSenderService` 优先使用模板，否则使用自定义内容
- 验证逻辑：至少提供模板ID或自定义内容之一
- 响应格式化：自动检测响应内容格式（JSON/XML）并格式化
- 格式检测逻辑：
  - 以 `{` 或 `[` 开头识别为 JSON
  - 以 `<` 开头识别为 XML
  - 格式化失败时返回原内容
- 日志记录区分模板模式和自定义模式

## 测试建议

### 模板模式测试
1. 创建多个不同类型的报文模板
2. 测试协议 Tab 切换是否流畅
3. 验证请求预览是否正确渲染
4. 测试发送成功和失败的响应展示
5. 检查不同屏幕尺寸下的布局适配

### 自定义模式测试
1. 选择"自定义输入"选项
2. 输入 JSON 格式报文，测试发送
3. 输入 XML 格式报文，测试发送
4. 测试空内容时的验证提示
5. 测试模板模式和自定义模式的切换

### 综合测试
1. 模板模式发送成功后，切换到自定义模式
2. 自定义模式发送后，切换回模板模式
3. 验证响应结果是否正确保留
4. 测试不同协议的参数配置
5. 测试 JSON 响应的格式化和高亮显示
6. 测试 XML 响应的格式化和高亮显示
7. 测试纯文本响应的显示
8. 验证格式化失败时是否正确降级显示原内容
