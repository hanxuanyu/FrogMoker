# 通用预览组件 - 紧凑布局设计

## 设计理念

**一眼看全，紧凑高效**

简单参数使用紧凑的徽章式布局，左右依次堆叠，充分利用横向空间。用户可以一眼看到所有参数配置，无需上下滚动。

## 布局对比

### 之前的布局（垂直列表）

```
┌─────────────────────────────────────┐
│ 请求参数                             │
├─────────────────────────────────────┤
│ WebSocket 地址 *                     │
│ ws://localhost:8080/ws/echo         │
│ WebSocket 服务器地址，支持 ws:// ... │
│                                     │
│ 连接超时                             │
│ 5000                                │
│ 建立连接的超时时间（毫秒）            │
│                                     │
│ 等待响应                             │
│ [是]                                │
│ 发送消息后是否等待服务器响应          │
│                                     │
│ 期望响应数                           │
│ 1                                   │
│ 期望接收的响应消息数量                │
└─────────────────────────────────────┘
```

**问题：**
- ❌ 占用大量垂直空间
- ❌ 需要滚动才能看到所有参数
- ❌ 描述信息重复（左侧已有）
- ❌ 视觉密度低

### 现在的布局（紧凑徽章）

```
┌─────────────────────────────────────────────────────────────┐
│ 请求参数                                                     │
├─────────────────────────────────────────────────────────────┤
│ ┌──────────────────────────────┐ ┌──────────────────┐      │
│ │ WebSocket 地址* : ws://...   │ │ 连接超时 : 5000  │      │
│ └──────────────────────────────┘ └──────────────────┘      │
│                                                             │
│ ┌──────────────┐ ┌──────────────────┐                      │
│ │ 等待响应 : 是 │ │ 期望响应数 : 1   │                      │
│ └──────────────┘ └──────────────────┘                      │
└─────────────────────────────────────────────────────────────┘
```

**优势：**
- ✅ 充分利用横向空间
- ✅ 一眼看到所有参数
- ✅ 无需滚动
- ✅ 视觉密度高
- ✅ 简洁清晰

## 视觉设计

### 参数徽章样式

```css
/* 徽章容器 */
inline-flex items-center gap-1.5
px-3 py-1.5
rounded-md border
bg-muted/50
text-xs

/* 参数名 */
font-medium text-muted-foreground

/* 分隔符 */
text-muted-foreground/50

/* 参数值 */
font-mono
```

### 视觉层次

1. **协议信息卡片**（顶部）
   - 协议标识 Badge
   - 协议名称
   - 协议描述

2. **简单参数卡片**（中部）
   - 标题："请求参数"
   - 徽章网格：左右堆叠，自动换行

3. **复杂参数卡片**（底部）
   - 每个参数独立卡片
   - 参数名 + 类型 Badge
   - 内容区域（表格或代码编辑器）

## 实现细节

### 紧凑参数渲染

```typescript
<div className="flex flex-wrap gap-2">
  {groupedParams.simple.map((param) => {
    const value = params[param.name] || param.defaultValue || ""
    const displayValue = renderCompactParamValue(param, value)
    if (!displayValue) return null

    return (
      <div
        key={param.name}
        className="inline-flex items-center gap-1.5 px-3 py-1.5 rounded-md border bg-muted/50 text-xs"
      >
        <span className="font-medium text-muted-foreground">
          {param.label}
          {param.required && <span className="text-destructive ml-0.5">*</span>}
        </span>
        <span className="text-muted-foreground/50">:</span>
        <span className="font-mono">{displayValue}</span>
      </div>
    )
  })}
</div>
```

### 值简化处理

```typescript
const renderCompactParamValue = (param, value) => {
  switch (param.paramType) {
    case "BOOLEAN":
      return value === "true" ? "是" : "否"
    case "SELECT":
    case "TEXT":
    case "NUMBER":
    default:
      return value
  }
}
```

## 响应式行为

### 自动换行

使用 `flex-wrap` 实现自动换行：
- 宽度充足时：参数横向排列
- 宽度不足时：自动换行到下一行
- 始终保持紧凑布局

### 间距控制

- 徽章之间：`gap-2`（8px）
- 徽章内部：`gap-1.5`（6px）
- 卡片之间：`space-y-4`（16px）

## 使用场景

### WebSocket 协议示例

**配置：**
```
url: ws://localhost:8080/ws/echo
message: Hello, WebSocket!
connectTimeout: 5000
responseTimeout: 10000
waitForResponse: true
expectedMessages: 1
```

**预览效果：**

```
┌─────────────────────────────────────────────────────────────┐
│ [WEBSOCKET] WebSocket 客户端                                 │
│ 通过 WebSocket 协议建立连接并发送消息，支持实时双向通信      │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│ 请求参数                                                     │
├─────────────────────────────────────────────────────────────┤
│ [WebSocket 地址* : ws://localhost:8080/ws/echo]             │
│ [连接超时 : 5000] [响应超时 : 10000]                        │
│ [等待响应 : 是] [期望响应数 : 1]                             │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│ 消息内容 * [TEXTAREA]                                        │
├─────────────────────────────────────────────────────────────┤
│ Hello, WebSocket!                                           │
└─────────────────────────────────────────────────────────────┘
```

### HTTP 协议对比

HTTP 协议使用专用预览组件 `HttpRequestPreview`，提供更详细的 HTTP 请求信息：
- 请求行（方法 + URL）
- 请求头表格
- 查询参数表格
- 请求体（语法高亮）

而 WebSocket 等其他协议使用通用预览，同样清晰、紧凑。

## 设计优势

### 1. 信息密度高

紧凑布局使得更多信息可以在一屏内展示，减少滚动操作。

### 2. 视觉扫描快

横向排列的徽章更符合人眼的扫描习惯，可以快速定位目标参数。

### 3. 空间利用率高

充分利用横向空间，避免垂直空间浪费。

### 4. 层次清晰

通过卡片分组，简单参数和复杂参数层次分明。

### 5. 一致性好

所有协议使用统一的预览风格，用户体验一致。

## 最佳实践

### 1. 参数命名

使用简短、清晰的参数标签：
- ✅ "连接超时"
- ❌ "建立连接的超时时间（毫秒）"

### 2. 参数分组

合理使用参数类型：
- 简单配置：TEXT、NUMBER、BOOLEAN、SELECT
- 复杂内容：TEXTAREA、MAP、ARRAY

### 3. 默认值

提供合理的默认值，减少空白徽章。

### 4. 参数联动

通过联动隐藏不相关参数，保持预览简洁。

## 技术细节

### Tailwind CSS 类

```typescript
// 容器
"flex flex-wrap gap-2"

// 徽章
"inline-flex items-center gap-1.5 px-3 py-1.5 rounded-md border bg-muted/50 text-xs"

// 参数名
"font-medium text-muted-foreground"

// 分隔符
"text-muted-foreground/50"

// 参数值
"font-mono"

// 必填标记
"text-destructive ml-0.5"
```

### 响应式断点

组件自动适应容器宽度，无需手动设置断点：
- 使用 `flex-wrap` 自动换行
- 使用 `gap-2` 保持一致间距
- 使用 `inline-flex` 自适应内容宽度

## 未来增强

可以考虑的增强功能：

1. **参数分类**
   - 按功能分组（连接配置、消息配置、超时配置）
   - 使用分隔线或子标题

2. **交互功能**

   - 悬停显示完整描述

3. **视觉优化**
   - 不同类型参数使用不同颜色
   - 必填参数使用强调色

4. **导出功能**
   - 导出为 JSON
   - 导出为命令行参数

## 总结

紧凑布局设计大幅提升了通用预览组件的可用性：

- **空间效率**：从垂直列表到横向网格，空间利用率提升 3-4 倍
- **视觉效率**：一眼看全所有参数，无需滚动
- **信息效率**：移除重复描述，只保留关键信息

这种设计使得通用预览组件不仅能用，而且好用，为所有协议提供了专业、高效的预览体验。
